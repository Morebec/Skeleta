Columns

The Columns is a container that displays its elements horizontally and which allows spacing them evenly on the x axis. The columns container will shrink the elements' widths as much as they allow it to make them fit, until it no longer can and will overflow. The columns is an interresting one as it is non-responsive by design. It allows some advanced use cases where you'd want to show more or less of the content depending on the width of the screen. For instance, some hero layouts having a call out in one columns and an image or media in another one rely on this very feature to allow hiding the image or showing less of it depending on the screen width.

Stack

The stack is a container that stacks its elements vertically and which allows spacing them evenly on the y axis. Unless specified otherwise, the stack container will expand the width of its elements to take the full width of the stack container.

Switcher

The switcher is a responsive layout that will switch from a columns-like layout to a stack-like layout at a specific "breakpoint", i.e. a point where horizontally the space is considered insufficient to display the elements side by side. By default, if the width of the switcher is half a line-length (120ch) or less it will switch to a stack-like layout.

Switcher switching points

The switcher supports different breakpoint modifiers based on the unit scale, and rem values (from 10 to 100 in increments of 5). By default, if the width of the switcher is half a line-length (120ch) or less it will switch to a stack-like layout.
Here are the different classes that can be used, their naming convention follow a mobile-first approach:

  • Unit scale: unstack-from-unit{-5..10}
  • rem values: unstack-from-{10..100 in 5 increments}rem

Sidebar

Represents a layout that displays two columns side by side with a shared height, where one is narrower (the sidebar) and another filling up the remaining horizontal space (the main area). When in a narrow environment the two columns stack on top of each other similarly to the Switcher. It is named after the classic sidebar layout that exibits this behaviour. The first element is always considered the sidebar, while the second the width filling one.
